# Commit Message 규칙

## 1. 전체 길이 제한
- **최대한 짧고 명확하게 작성**
- **한 줄(subject)** 은 **영문 기준 50자, 한글 기준 30자 이내**를 권장
- 여러 줄을 쓰더라도 **전체 3줄 이내**로 설명 정리

## 2. 기본 형식
```text
<type>: <subject>

<optional body>
```

- **type** 예시: `feat`, `fix`, `refactor`, `chore`, `docs`, `style`, `test`
- **subject**: 이번 커밋으로 “무엇이 바뀌었는지”를 한 문장으로 요약

## 3. 좋은 예시
- `feat: hero 섹션 타이포그래피 축소`
- `fix: brand 버튼 텍스트 가독성 개선`
- `chore: unused rocket 컴포넌트 제거`

## 4. 피해야 할 예시
- `update`, `fix`, `수정`처럼 **의미가 모호한 단어만 있는 메시지**
- 한 줄에 **너무 많은 맥락(그리고, 그리고, 그리고…)** 을 넣는 메시지

## 5. Body(선택)
- 설명이 더 필요할 때만 작성
- **최대 2줄** 안에서 핵심만 요약
- 예: 왜 이렇게 수정했는지, 중요한 결정 사항 등
